# Change Log (2023)
---

This document records the update content of each release of Guance, including DataKit, Guance best practices, Guance integration documentation, and Guance.

## December 28, 2023

### Guance Update

- Monitoring:
    - Added **Smart User Visit Detection**: Supports performance anomaly detection and error analysis for websites/APPs. Relevant detection metrics include LCP, FID, CLS, etc.
    - **Infrastructure Survival Detection V2**: Introduced V2 version of infrastructure survival detection, optimized detection logic, and resolved several triggering issues related to V1 and process detection.
- RUM (User Access Monitoring):
    - Added **Public Network DataWay** for receiving RUM data without installing collectors; Web applications support direct upload of SourceMap in the console, allowing multiple files to be uploaded based on different versions and environments.
    - **Error Viewer > Error Details**: Added a **Formatted** error information page that displays error messages parsed from SourceMap files, supporting switching between formatted and original error message formats. If no SourceMap file is configured or does not match after uploading, only the original format error message will be displayed.
- APM (Application Performance Monitoring):
    - Optimized **Service Map**: Adjusted display styles for upstream/downstream, nodes, and service names.
    - Optimized **Service Details**: Bound performance view dashboard to display current service performance metrics, also supports customizing user views with the same name to cover updates and supports drill-down analysis.
    - Optimized **Trace Detail** Flame Graph and Waterfall Chart: Added execution time and percentage display to flame graphs; waterfall charts support relative and absolute time switch display.
- Chart Optimization:
    - Added **Display Columns** configuration for **Time Series Table Charts**: Supports user-defined query data columns, customizable column names; users can manually adjust column width in edit mode, saving settings to display list information according to column width; supports dragging to adjust the order of displayed columns.
    - Added **Value Variables** for chart links to use as jump links.
- Dashboard:
    - Added **Home Dashboard**: Supports workspace configuration of home dashboards.
- Infrastructure:
    - Added **Online Host Only Display** switch filter to Host Viewer: Optimized logic for obtaining online host lists, supports selecting whether to list hosts that have reported data within the last 10 minutes.


### Guance Deployment Plan Update

- Supported embedding Guance pages in external platforms using URL parameters to hide left and top menu bars and floating operations.
- Supported embedding Guance dashboards or viewer pages via iframe in external platforms.

### New Integrations

- Added integration for **Kubecost**

## December 14, 2023

### Guance Update

- Monitoring:
    - Added **Smart Monitoring**:
        - **Host Smart Detection**: Supports CPU spike/drop/interval increase detection, memory spike/drop/interval increase detection.
        - **Log Smart Detection**: Supports log count spike/drop detection and error log spike detection.
        - **Application Smart Detection**: Supports request count spike/drop, error request count spike, request latency spike/drop/interval increase detection.
    - **Mutation Detection**: Added prerequisite conditions for trigger rules, initiating mutation detection when monitored values meet these conditions, improving monitoring accuracy.
- Paid Plans and Billing:
    - Added **Data Forwarding Usage Analysis** list: View all data forwarding rules and quantities in the current workspace. If the forwarding rule is for backup logs to Guance, it shows corresponding retention policies. Also supports time control filtering for historical log backup overall situation, making it convenient to quickly view historical log backup situations.
- Events:
    - Added **Smart Monitoring Event Viewer**: Can view all event information generated by smart monitoring.
- Logs:
    - **Data Access** added [Show Only Rules Related to Me] configuration: When enabled, only access rules affecting the current account's data queries are shown.
    - **Logs > Viewer**: Added data access rule restriction prompts.
- Infrastructure > **Containers > Kubernetes**:
    - Added Kubernetes analysis dashboard: Container object analysis dashboard to help understand various infrastructure situations in Kubernetes from different dimensions.
    - Container > Viewer: UI optimization.
- Metrics:
    - **Metrics > Viewer**:
        - Added [Query Tool] mode for table charts: Returns results consistent with **Shortcuts > Query Tool**; if multiple queries are added, defaults to returning the first query result; this mode supports exporting up to 1000 query records.
        - Added [Copy] function for query conditions, optimizing page experience.
        - Optimized query components: Simple queries' [by Grouping] support free drag-and-drop adjustment of order.
- Viewer:
    - Adjusted all viewers to hide trend chart display in analysis mode.
- SSO:
    - Optimized SSO user workspace listing logic, fixing automatic logout due to lack of access permissions.

### Guance Deployment Plan Update

- Added **Independent Alarm Notification Sending** configuration: When enabled, each event sends an alarm notification individually when the alarm policy selects the [Do Not Aggregate Alarms] option, helping efficiently manage alarm notifications.
- Added **Event Link Login-Free Viewing** configuration: When enabled, all event-related links in alert notifications sent by workspaces can be accessed without logging in. If disabled, historical login-free links become invalid, ensuring data security.
- Optimized [Scheduled Reports]: Supports customizable query range configuration, increasing the upper limit from 30 days to 360 days to meet extended scheduled report needs.

### New/Updated Integrations

- Updated **Aliyun Kafka**
- Updated **Dameng**

## November 9, 2023

### Guance Update

- Monitors > **Alert Policy Management**:
    - Notification Configuration: Changed to configure single or multiple notification targets for individual anomaly levels.
    - Recovery Notification Logic Change: When previously sent anomaly alerts recover, Guance sends recovery notifications to corresponding notification targets.
    - Added [**Alert Aggregation**]: Supports configuring alert notification aggregation based on rules, with two aggregation methods:
        - No Aggregation: Under this mode, every 20 seconds merge into one alert notification sent to the corresponding notification targets.
        - Rule Aggregation: Choose four aggregation rules and send alert notifications based on the aggregation cycle.
    - Added [Aggregation Cycle]: In the rule aggregation mode, new events within 1-30 minutes will be aggregated into notifications sent, and new events exceeding the aggregation cycle will be aggregated into new alert notifications.
- Management > **Data Forwarding**: Supports custom time range queries, allowing selection of multiple dates and defining start and end times.
