# Add SAML SSO Login Configuration

---

<br />**POST /api/v1/sso/saml_create**

## Overview
Enable a SAML configuration.

## Body Request Parameters

| Parameter Name | Type   | Required | Description                                                                                           |
|:--------------|:-------|:--------|:-----------------------------------------------------------------------------------------------------|
| type          | string | Y       | Type<br>Example: <br>Allow null: False <br>Optional values: ['saml', 'oidc'] <br>                   |
| idpData       | string | Y       | XML document content (required for SAML type)<br>Example: <br>Allow null: False <br>                |
| config        | json   |         | Access configuration table (required for OIDC type)<br>Example: <br>Allow null: False <br>          |
| emailDomains  | array  | Y       | Email domains<br>Example: ['jiagouyun.com', 'guance.com'] <br>                                      |
| idpName       | string | Y       | Provider<br>Example: Default provider <br>Maximum length: 64 <br>$matchRegExp: [a-zA-Z_一-龥-]* <br>|
| role          | string | Y       | Role<br>Allow null: False <br>Optional values: ['general', 'readOnly'] <br>                          |
| remark        | string |         | Remarks<br>Allow null: False <br>Allow empty string: True <br>                                       |
| tokenHoldTime | integer|         | Token retention duration in seconds, default value 14400<br>Allow null: False <br>Allow empty string: False <br>$minValue: 1800 <br>$maxValue: 86400 <br> |
| tokenMaxValidDuration | integer |         | Maximum validity period of the token in seconds, default value 604800<br>Allow null: False <br>Allow empty string: False <br>$minValue: 86400 <br>$maxValue: 604800 <br> |

## Additional Parameter Explanation

*OIDC Type config Configuration Explanation*

--------------
When `type='oidc'`, the `config` field takes effect. The data structure information is as follows:
<br/>
1. config Field Explanation

| Parameter Name | Type  | Required | Default Value | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|----------------|-------|----------|---------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| modeType       | enum  |          | easy          | Configuration file editing mode. Optional values:<br/>easy: Simple UI editing mode. In this mode, users only need to configure the basic data required by the OIDC interaction protocol; all other data are default.<br/>expert: Expert configuration file mode, which requires users to upload OIDC configuration files.
