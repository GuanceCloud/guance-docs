# 用户访问智能检测
---


利用智能检测算法，系统能够智能监测用户访问网站或APP时的性能异常和错误突增情况。通过分析检测指标，系统将自动进行异常分析，并在必要时触发告警。

性能分析指标包括：

- LCP（最大内容绘制时间大于或等于 2.5 秒）；     
- INP（交互时间大于或等于 200 毫秒）；  
- FID（首次输入延迟大于或等于 100 毫秒）；    
- CLS（累积布局偏移大于 0.01）。

基于这些性能分析指标，系统将统计检测区间内受影响用户的占比，并根据预设的告警级别阈值进行告警：

- 严重（Critical）级别：受影响用户占比超过 76.2%；  
- 警告（Warning）级别：受影响用户占比超过 47.4%；  
- 错误（Error）级别：受影响用户占比超过 59.8%。  

错误检测指标包括：

- 会话错误数；  
- 页面错误数；  
- 动作错误数。 



## 应用场景

支持监测包括 Web 和 APP 在内的应用类型的指标数据异常。


## 检测配置 {#config}

![](../img/intelligent-detection15.png) 

1. 定义监控器名称；

2. 选择检测频率：即检测规则的执行频率，固定 60 分钟；

3. 选择检测范围：通过字段过滤要检测的数据范围，若不添加筛选，检测所有应用的数据。



## 查看事件

监控器会获取最近 60 分钟的检测应用程序服务对象指标信息，识别出现异常情况时，会生成相应的事件，在[**事件 > 智能监控**](../../events/inte-monitoring-event.md)列表可查看对应异常事件。


### 事件详情页

点击**事件**，可查看智能监控事件的详情页，包括事件状态、异常发生时间、异常名称、分析报告、告警通知、历史记录和关联事件。

* 点击右上角的**跳转到监控器**，可查看调整[智能监控器](index.md)；

* 点击右上角的**导出**按钮，支持选择**导出 JSON 文件**与**导出 PDF 文件**，从而获取当前事件所对应的所有关键数据。
