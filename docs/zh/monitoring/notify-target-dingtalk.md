# 钉钉机器人 

为了适配最新的钉钉机器人模式，需先在钉钉开发平台创建企业内部应用机器人。

> 更多详情，可参考 [企业内部应用机器人的创建和安装](https://open.dingtalk.com/document/orgapp/overview-of-development-process)。

???- warning "钉钉机器人新旧区别"

    - 钉钉平台：创建机器人从**群管理**直接创建变更为在**开发平台创建应用**；   
    - <<< custom_key.brand_name >>>：最新的钉钉机器人密钥配置为非必填项。

## 创建企业内部应用机器人

首先向您的组织申请钉钉开发平台**开发者权限**。

### 创建应用
 
1. 选择**应用开发 > 钉钉应用 > 创建应用**，单击创建应用；   
2. 应用创建完成后，点击右侧**应用详情**，前往机器人配置页面填写相关配置；
3. 在目标群里点击**添加机器人**，在企业机器人列表中选择新创建的应用机器人；   
4. 获取机器人 Webhook 地址，在机器人管理找到新创建的应用机器人，点击查看详情，复制 Webhook 地址。

???+ warning "注意"

    此处使用机器人仅用于接收信息，不存在交互，消息接收模式配置可任意选择，HTTP 模式下地址可以为空。

## 回到钉钉机器人配置页面

1. 钉钉群组成功添加机器人之后，在机器人配置详情中可查询该机器人的**加签**密钥和 **Webhook** 地址。

2. 输入配置信息，包括自定义的通知对象名称、密钥和 Webhook 地址。

<img src="../img/10_inform_03.png" width="60%" >


## 在<<< custom_key.brand_name >>>侧配置

1. 回到配置钉钉机器人为通知对象的页面；
2. 填入在上述步骤获取的密钥和 **Webhook** 地址。
