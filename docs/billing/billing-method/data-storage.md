# 数据存储策略
---

随着使用的应用程序的增加，数据观测需求范围的扩大，对数据的存储时长的需求也会随之调整。观测云支持商业版拥有者变更空间内的数据存储策略，在观测云工作空间「管理」-「基本设置」，点击「变更」后，选择所需的数据存储时长，点击「确定」即可变更当前工作空间内的数据存储时长。

当前观测云共有免费版、商业版、独占合作版三种产品版本供您选择。针对上述不同的版本对应的数据存储策略选择区间也不尽相同。免费版数据存储策略可参考文档 [免费开始](../../billing/free-start.md) 。

| 数据存储类型 | **商业版** |
| --- | --- |
| 指标 | 3天 / 7天（默认）/ 14天 / 30天 / 180天 / 360天 |
| 日志 | 7天 / 14天（默认）/ 30天 / 60天 |
| 备份日志 | 180天（默认）/ 360天 / 720天 |
| 事件 | 14天（默认）/ 30天 / 60天 |
| 安全巡检 | 90天（默认）/ 180天 / 360天 |
| 应用性能 Trace | 3天 / 7天（默认）/ 14天 |
| 用户访问 PV | 3天 / 7天（默认）/ 14天 |

**注意：**

1. 只允许商业版工作空间拥有者进行此操作
1. 商业版拥有者可按照需求变更指标、日志、备份日志、事件、应用性能、用户访问、安全巡检等数据存储时长
1. 存储策略变更后，新的保存策略即时生效，旧的存储策略里面的数据不会立即删除，在策略滚动删除旧数据之前仍然存在这部分计费。
   - 数据保存时间缩短：假设应用性能数据从14天调整为7天，以当天调整的时间为分界线，在此之前的数据仍然按照14天存储策略计算费用，在此之后的数据会生成新的索引并按照新的7天存储策略计算费用；
   - 数据保存时间变长：假设应用性能数据从7天调整为14天，以当天调整的时间为分界线，在此之前的数据仍然按照7天存储策略计算费用，在此之后的数据会生成新的索引并按照新的14天存储策略计算费用。

更多数据存储策略变更说明，可参考文档 [观测云 ES多租户生命周期管理实践](../../billing/billing-method/es-life-cycle.md) 。
